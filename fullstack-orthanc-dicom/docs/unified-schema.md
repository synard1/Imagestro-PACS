# Unified Order & Accession Schema

Canonical schema for single-gate ordering via Order Management, with accession number generated by Accession API. Matches SIMRS payload fields.

## SIMRS Payload (canonical)
```
{
  "modality": "CT",
  "procedure_code": "CTABDOMEN",
  "procedure_name": "CTABDOMEN 0001",
  "scheduled_at": "2025-10-22T09:30:00Z",
  "patient_national_id": "1234567890123456",
  "patient_name": "John Doe",
  "gender": "male",
  "birth_date": "1990-01-01",
  "medical_record_number": "12345678",
  "ihs_number": "123456789",
  "registration_number": "RJ2025102300001"
}
```

## Orders Table (order-management)
- `id` UUID PK
- `order_number` VARCHAR(50) unique
- `accession_number` VARCHAR(50) unique (from accession-api)
- `modality` VARCHAR(10)
- `procedure_code` VARCHAR(50)
- `procedure_name` VARCHAR(200)
- `scheduled_at` TIMESTAMPTZ
- `patient_national_id` VARCHAR(16) NOT NULL
- `patient_name` VARCHAR(200) NOT NULL
- `gender` VARCHAR(10)
- `birth_date` DATE
- `medical_record_number` VARCHAR(50)
- `ihs_number` VARCHAR(50)
- `registration_number` VARCHAR(50)
- `status` VARCHAR(20) DEFAULT 'CREATED'
- `created_at` TIMESTAMPTZ DEFAULT now()
- `updated_at` TIMESTAMPTZ DEFAULT now()

## Accessions Table (accession-api)
- `id` UUID PK
- `accession_number` VARCHAR(50) unique NOT NULL
- `modality` VARCHAR(10)
- `procedure_code` VARCHAR(50)
- `procedure_name` VARCHAR(200)
- `scheduled_at` TIMESTAMPTZ
- `patient_national_id` VARCHAR(16) NOT NULL
- `patient_name` VARCHAR(200) NOT NULL
- `gender` VARCHAR(10)
- `birth_date` DATE
- `medical_record_number` VARCHAR(50)
- `ihs_number` VARCHAR(50)
- `registration_number` VARCHAR(50)
- `issuer` TEXT
- `status` VARCHAR(20) DEFAULT 'issued'
- `created_at` TIMESTAMPTZ DEFAULT now()
- `updated_at` TIMESTAMPTZ DEFAULT now()

## Behavior
- Order Management:
  - Accepts SIMRS payload at `POST /orders/create`.
  - Calls Accession API `POST /accession/create` with the same payload to obtain `accession_number`.
  - Persists order with returned `accession_number`.

- Accession API:
  - Generates `accession_number`.
  - Persists accession record mirroring the payload.
  - Returns `{ accession_number, id, issuer }` and echoes relevant fields.

## Notes
- `ihs_number` validation can be disabled to permit numeric-only values.
- `scheduled_at` stored as TIMESTAMPTZ; string ISO-8601 expected in requests.
- `gender` accepts `male|female|other|unknown` canonical values.
- DB indexes: unique on `accession_number`; optional indexes on `patient_national_id`, `registration_number`.
