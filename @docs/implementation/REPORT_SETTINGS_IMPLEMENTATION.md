# Report Settings Implementation - Complete Guide

## Overview
Customizable report header and footer settings for PDF export with logo upload, color customization, and text configuration.

**Status**: ✅ Complete  
**Date**: 2025-12-29  
**Feature**: Week 8 Day 1 Enhancement

---

## Features Implemented

### 1. Header Customization ✅
- **Hospital Logo**: Upload and display custom logo
- **Hospital Information**: Name, address, phone, email
- **Colors**: Custom background and text colors
- **Toggle**: Enable/disable header
- **Logo Size**: Adjustable width and height

### 2. Footer Customization ✅
- **Three Text Sections**: Left, center, right
- **Page Numbers**: Automatic with placeholders `{page}` and `{total}`
- **Timestamp**: Optional generation timestamp
- **Font Size**: Adjustable (6-12pt)
- **Text Color**: Custom color picker
- **Toggle**: Enable/disable footer

### 3. Report Settings ✅
- **Report Title**: Customizable title text
- **Font Sizes**: Title, section, and content
- **Page Margins**: Top, right, bottom, left (10-30mm)

### 4. Settings Management ✅
- **Save/Load**: localStorage persistence
- **Export/Import**: JSON file format
- **Reset**: Restore default settings
- **Preview**: Real-time preview in PDF

## API Reference

### `reportSettingsService.js`

#### `getReportSettings()`
Get current report settings from localStorage.

**Returns:** Settings object with header, footer, and report config

**Example:**
```javascript
import { getReportSettings } from './services/reportSettingsService';

const settings = getReportSettings();
console.log(settings.header.hospitalName);
```

---

#### `saveReportSettings(settings)`
Save report settings to localStorage.

**Parameters:**
- `settings` (Object): Complete settings object

**Returns:** `{ success: boolean, error?: string }`

**Example:**
```javascript
import { saveReportSettings } from './services/reportSettingsService';

const result = saveReportSettings({
  header: { enabled: true, hospitalName: 'New Hospital' },
  footer: { enabled: true },
  report: { title: 'RADIOLOGY REPORT' }
});

if (result.success) {
  console.log('Settings saved!');
}
```

---

#### `resetReportSettings()`
Reset settings to default values.

**Returns:** `{ success: boolean }`

**Example:**
```javascript
import { resetReportSettings } from './services/reportSettingsService';

resetReportSettings();
```

---

#### `uploadLogo(file)`
Upload and process logo image.

**Parameters:**
- `file` (File): Image file (PNG, JPG, max 2MB)

**Returns:** Promise resolving to `{ dataUrl, width, height }`

**Example:**
```javascript
import { uploadLogo } from './services/reportSettingsService';

const handleFileUpload = async (e) => {
  const file = e.target.files[0];
  try {
    const result = await uploadLogo(file);
    console.log('Logo uploaded:', result.dataUrl);
  } catch (error) {
    console.error('Upload failed:', error.message);
  }
};
```

---

#### `exportSettings()`
Export settings as JSON file.

**Example:**
```javascript
import { exportSettings } from './services/reportSettingsService';

exportSettings();  // Downloads report-settings-{timestamp}.json
```

---

#### `importSettings(file)`
Import settings from JSON file.

**Parameters:**
- `file` (File): JSON file with settings

**Returns:** Promise resolving to imported settings object

**Example:**
```javascript
import { importSettings } from './services/reportSettingsService';

const handleImport = async (e) => {
  const file = e.target.files[0];
  try {
    const settings = await importSettings(file);
    console.log('Settings imported:', settings);
  } catch (error) {
    console.error('Import failed:', error.message);
  }
};
```

## PDF Generator Integration

### How Settings Are Applied

```javascript
// In pdfGenerator.js
import { getReportSettings } from './reportSettingsService';

export const generateReportPDF = (reportData, signatureData = null) => {
  // Load settings
  const settings = getReportSettings();
  
  // Apply header settings
  if (settings.header.enabled) {
    // Add logo
    if (settings.header.logo) {
      doc.addImage(settings.header.logo, 'PNG', x, y, w, h);
    }
    
    // Add hospital info
    doc.text(settings.header.hospitalName, x, y);
  }
  
  // Apply footer settings
  if (settings.footer.enabled) {
    // Replace placeholders
    const footerText = settings.footer.rightText
      .replace('{page}', pageNum)
      .replace('{total}', totalPages);
    
    doc.text(footerText, x, y);
  }
  
  // Apply report settings
  doc.setFontSize(settings.report.titleFontSize);
  doc.text(settings.report.title, x, y);
};
```

## Logo Upload

### Specifications
- **Formats**: PNG, JPG, GIF
- **Max Size**: 2MB
- **Auto Resize**: Images larger than 200x200px are automatically resized
- **Storage**: Base64 encoded in localStorage
- **Display**: Proportional scaling in PDF

### Upload Process
1. User selects image file
2. File is validated (type, size)
3. Image is loaded and resized if needed
4. Converted to base64 data URL
5. Stored in settings
6. Preview shown immediately

## Troubleshooting

### Issue: Logo not showing in PDF
**Solution**: 
- Check logo file size (< 2MB)
- Verify logo is uploaded successfully
- Check browser console for errors
- Try re-uploading logo

### Issue: Settings not persisting
**Solution**:
- Check localStorage is enabled
- Clear browser cache
- Check for localStorage quota errors
- Try export/import as backup

### Issue: Colors not applying
**Solution**:
- Use hex color format (#RRGGBB)
- Check color picker value
- Verify settings are saved
- Reload page after save

### Issue: Page numbers not showing
**Solution**:
- Enable "Show Page Numbers" checkbox
- Use placeholders `{page}` and `{total}` in right text
- Verify footer is enabled
- Check footer text is not empty

## Security Considerations

### Data Storage
- Settings stored in localStorage (client-side only)
- Logo images stored as base64 (no server upload)
- No sensitive data in settings
- Export/import uses JSON (no executable code)

### Best Practices
- Validate logo file type and size
- Sanitize text inputs
- Limit logo dimensions
- Check localStorage quota
- Handle errors gracefully

### Example 3: Upload Logo
```javascript
import { uploadLogo, getReportSettings, saveReportSettings } from './services/reportSettingsService';

const handleLogoUpload = async (e) => {
  const file = e.target.files[0];
  try {
    const result = await uploadLogo(file);
    const settings = getReportSettings();
    settings.header.logo = result.dataUrl;
    settings.header.logoWidth = result.width;
    settings.header.logoHeight = result.height;
    saveReportSettings(settings);

  } catch (error) {
    alert('Upload failed: ' + error.message);
  }
};
```
